# 설정 화면 개선 PRD

## 1. 개요

현재 placeholder 상태인 설정 페이지(`/settings`)를 실제 운영 설정을 조회할 수 있는 화면으로 개선한다. 서버의 현재 설정값(config.yaml + 환경변수 반영 결과)을 읽기 전용으로 표시한다.

## 2. 배경 및 목적

- **현재 상태**: 설정 페이지는 "준비 중" placeholder만 표시
- **목표**: 서버 설정값을 한눈에 확인하고, 시스템 상태를 모니터링할 수 있는 운영 대시보드 제공
- **범위 제한**: 이번 개선은 **읽기 전용** — 설정 변경(쓰기)은 추후 고려

## 4. 변경 범위

### 4.1 백엔드: Config 조회 API (신규)

#### `GET /api/config`
서버의 현재 적용된 설정값을 반환한다 (민감 정보 제외).

```json
{
  "server": {
    "api_port": 8080
  },
  "receiver": {
    "grpc_port": 4317,
    "http_port": 4318
  },
  "processor": {
    "batch_size": 1000,
    "flush_interval": "2s",
    "queue_size": 10000,
    "drop_on_full": true
  },
  "storage": {
    "path": "./data/apm.db",
    "retention_days": 7
  }
}
```

#### `GET /api/system`
서버 런타임 정보를 반환한다.

```json
{
  "version": "v0.1.0-alpha",
  "go_version": "go1.22.4",
  "os": "darwin",
  "arch": "arm64",
  "uptime_seconds": 223920,
  "data_dir_size_bytes": 130547712
}
```

### 4.2 프론트엔드

#### Settings.tsx 전면 개편
- placeholder → 실제 설정 조회 화면
- 4개 섹션 카드: 수집기 설정, 배치 프로세서, 스토리지, 시스템 정보
- 각 설정 항목을 key-value 형태의 읽기 전용 테이블로 표시
- 시스템 정보 섹션에 가동 시간, 데이터 디렉토리 크기 등 런타임 정보 포함

## 6. 디자인 가이드라인

- 각 섹션: `rounded-xl border border-slate-800 bg-[#0f172a]`
- 설정 항목: 좌측 레이블 (`text-slate-400 text-sm`) + 우측 값 (`text-slate-100 font-mono`)
- 포트 상태 표시: 녹색 점 + "수신 대기 중" (health check 기반)
- 보존 기간: `7일` 형식, 경고 색상은 3일 이하일 때 amber

## 7. 제외 범위

- 설정 변경(PATCH/PUT) — 읽기 전용으로만 제공
- config.yaml 파일 직접 편집 UI
- 환경변수 오버라이드 표시 (어떤 값이 env에서 왔는지 구분)
