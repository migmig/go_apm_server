# 긴급 장애 대응 및 원인 추적(Troubleshooting)을 위한 UI/UX 심층 분석 보고서

## 1. 개요
본 문서는 장애 상황에서 SRE 및 DevOps 엔지니어가 APM 도구를 사용하여 문제를 격리하고 원인을 파악하는 과정(Troubleshooting Workflow)에서의 사용성을 심층 분석하고, 이를 개선하기 위한 Phase 5 기획안을 제안합니다.

## 2. 사용자 페르소나 및 핵심 워크플로우
- **페르소나:** 장애 알람을 받고 즉시 투입된 SRE/On-call 엔지니어
- **목표:**
    1. "어느 서비스가 문제인가?" (격리)
    2. "무엇이 잘못되었는가?" (에러 로그 확인)
    3. "왜 느린가?" (병목 구간 파악)
- **핵심 워크플로우:** Dashboard (이상 감지) -> Trace List (지연/에러 요청 선택) -> Trace Detail (병목/오류 구간 확인) -> Logs (상세 원인 파악)

## 3. 현황 분석 및 문제점 (Pain Points)

### 3.1. Dashboard: 인지적 부하와 정보 우선순위
- **문제점:** 서비스별 성능 현황 테이블에서 '위급' 상태가 애니메이션으로 표시되지만, 많은 서비스가 나열될 경우 한눈에 들어오지 않음.
- **분석:** 긴박한 상황에서는 수치보다 '색상'과 '상태'의 가시성이 중요함. 현재 테이블 형태는 데이터 밀도가 높아 스캔 속도가 떨어짐.

### 3.2. Trace Detail (Waterfall): 병목 구간의 시각적 강조 부족
- **문제점:** 워터폴 차트에서 모든 스팬이 동일한 시각적 무게를 가짐. 100개 이상의 스팬이 있는 경우, 어느 구간이 전체 지연의 80%를 차지하는지 일일이 수치를 읽어야 함.
- **분석:** "Critical Path" 또는 전체 시간 대비 점유율이 높은 스팬을 자동으로 강조하는 기능이 부재함.

### 3.3. Trace-to-Log 연결성 (Correlation Gap)
- **문제점:** 특정 스팬에서 에러가 발생했을 때, 해당 스팬과 관련된 '로그'만 따로 모아보기 어려움. 현재는 전체 로그 화면으로 이동하여 Trace ID로 검색해야 함.
- **분석:** 컨텍스트 스위칭 비용이 발생함. 특정 스팬을 클릭했을 때 그 시점, 그 서비스의 로그로 즉시 연결되는 "Deep Link"가 필요함.

### 3.4. 필터링 및 시간 범위 조작의 경직성
- **문제점:** 특정 시간대(예: 장애 발생 시점 5분간)로 범위를 좁히는 조작이 번거로움.
- **분석:** 장애 분석은 특정 'Time Window'에 집중됨. 이를 위한 퀵 필터나 직관적인 시간 선택 UI 보강이 필요함.

## 4. Phase 5 개선 제안 (Action Plan)

### 4.1. [Quick Win] Trace Detail: 스팬별 로그 즉시 보기 (Log Context Jump)
- **내용:** Trace Detail의 사이드바(Metadata)에 "해당 스팬의 로그 보기" 링크 추가.
- **효과:** Trace ID와 Service Name이 포함된 로그 필터로 즉시 이동하여 분석 시간 단축.

### 4.2. [Visual] Waterfall: 병목 구간 하이라이팅
- **내용:** 전체 Trace Duration의 30% 이상을 차지하는 'Heavy Span'에 대해 별도의 테두리나 아이콘 표시.
- **효과:** 스크롤을 내리며 병목 지점을 즉시 육안으로 식별 가능.

### 4.3. [Observability] Dashboard: 서비스 헬스 카드 도입
- **내용:** 대시보드 상단에 '위급' 상태인 서비스들만 모아 보여주는 요약 섹션 추가.
- **효과:** 어떤 서비스가 죽었는지 들어오자마자 1초 만에 파악.

### 4.4. [Advanced] 로그 뷰어: Severity 분포 차트 추가
- **내용:** 로그 검색 결과 상단에 시간대별 에러/경고 발생 빈도를 보여주는 미니 차트 추가.
- **효과:** 장애 발생 시점의 로그 급증 패턴을 시각적으로 확인.

## 5. 결론
현재의 UI는 데이터 표현에는 충실하나, '긴급 상황'에서의 의사결정을 돕는 가이드 역할은 부족합니다. Phase 5에서는 데이터 간의 연결(Correlation)을 강화하고, 시각적 우선순위를 재조정하여 원인 추적 속도를 2배 이상 향상시키는 것을 목표로 합니다.
