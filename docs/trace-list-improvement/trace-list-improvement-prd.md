# [PRD] 요청 추적 화면(Trace List) 사용성 및 레이아웃 개선

## 1. 개요
현재 요청 추적 화면의 데이터 표기 방식과 인터페이스를 개선하고, 대화면 환경에서도 효율적으로 데이터를 확인할 수 있도록 레이아웃을 최적화하는 것을 목표로 합니다.

## 2. 주요 개선 사항

### 2.1 요청 시간 형식 변경 (한국식)
*   **현황:** `MMM dd, HH:mm:ss.SSS` 형식 (예: Mar 11, 14:30:15.123)
*   **변경:** `YYYY-MM-DD HH:mm:ss` 형식 (예: 2026-03-11 14:30:15)
*   **목적:** 국내 표준 날짜 표기법을 준수하여 직관적인 시간 파악 가능

### 2.2 요청 ID(Trace ID) 클릭 시 상세 이동
*   **현황:** 요청 ID가 단순 텍스트이며, 상세 화면 진입을 위해 우측 '상세 보기' 버튼만 사용 가능
*   **변경:** 요청 ID 텍스트에 링크를 적용하여 클릭 시 상세 페이지(`/traces/{trace_id}`)로 이동
*   **목적:** 데이터 접근성 향상 및 사용자 동선 최적화

### 2.3 수행 작업(Operation) 칸 최적화 및 툴팁 도입
*   **현황:** 긴 작업 명칭으로 인해 테이블 레이아웃이 가변적이며 가독성이 떨어짐
*   **변경:** 
    *   컬럼 너비를 적절히 제한하고 넘치는 텍스트는 말줄임표(`...`) 처리
    *   Hover 시 전체 텍스트를 확인할 수 있도록 브라우저 기본 `title` 속성 또는 툴팁 제공
*   **목적:** 테이블 구조의 안정성 확보 및 상세 정보 확인 편의성 증대

### 2.4 대화면 레이아웃 최적화
*   **현황:** 현재 페이지의 전체적인 너비가 제한되어 있어, 고해상도(QHD 이상) 모니터에서 좌우 여백이 과도하게 발생함
*   **변경:** 
    *   컨테이너의 최대 너비(`max-w-7xl` 등)를 확장하거나 제거하여 넓은 화면을 충분히 활용하도록 수정
    *   테이블 내 각 컬럼의 간격을 조절하여 시각적 답답함 해소
*   **목적:** 대화면 사용자에게 더 많은 정보를 한 화면에 쾌적하게 제공

## 3. 상세 기능 요구사항 (User Story)

| ID | 사용자 스토리 | 수용 기준 (Acceptance Criteria) |
| :--- | :--- | :--- |
| **F-1** | 사용자는 한국식 날짜 형식을 통해 요청 시간을 한눈에 파악하고 싶다. | `yyyy-MM-dd HH:mm:ss` 포맷 적용 (밀리초 제외) |
| **F-2** | 사용자는 요청 ID를 클릭하여 즉시 상세 페이지로 이동하고 싶다. | `trace_id` 텍스트에 `Link` 적용 및 호버 시 언더라인/컬러 강조 효과 |
| **F-3** | 사용자는 긴 수행 작업 명칭 때문에 테이블이 흐트러지는 것을 방지하고 싶다. | CSS `truncate` 적용 및 `title={trace.root_span}` 추가 |
| **F-4** | 사용자는 넓은 모니터에서 더 시원한 화면 구성을 보고 싶다. | 최상위 레이아웃 컨테이너의 너비 제한 완화 (`max-w-screen-2xl` 이상 권장) |
