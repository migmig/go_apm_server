# 프로젝트 가이드라인 (CLAUDE.md)

## 1. 작업 프로세스
모든 신규 기능 구현 및 개선 작업은 반드시 다음 단계를 준수해야 합니다.

1.  **PRD (Product Requirements Document):** 제품의 목표, 요구사항, 사용자 가치를 정의합니다. (`docs/` 하위 폴더에 저장)
2.  **Spec (Specification):** 기술적인 사양, UI/UX 상세, API 명세 등을 정의합니다.
3.  **Tasks:** 구체적인 구현 단계와 체크리스트를 작성합니다.

## 2. 테스트 원칙
- 모든 Task 진행 시 **테스트 수행은 필수**입니다.
- 단순 코드 수정을 넘어, 실제 환경에서의 동작을 반드시 검증해야 합니다.

## 3. 검증 방법 (Browser MCP 활용)
- 프론트엔드 UI 변경 사항이나 사용자 흐름(User Flow)은 **Browser MCP**를 사용하여 테스트를 진행해야 합니다.
- 테스트 시나리오에는 다음이 포함되어야 합니다:
    - 기능 정상 작동 여부 (Click, Link, Data Display 등)
    - 반응형 레이아웃 및 대화면 최적화 확인
    - 에러 상황 및 예외 처리 확인
